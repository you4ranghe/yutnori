## 완료 기록

### 1단계 · 높음 — 2026-07-22

**1. 실시간 연결 끊김 처리**

- 방 채널 구독에 `onChannelStatus()` 를 붙여 `CHANNEL_ERROR`·`TIMED_OUT`·`CLOSED` 를 감지합니다.
- 끊기면 화면 위에 **📡 연결이 끊겼어요 — 다시 잇는 중…** 표시가 뜨고, 회복되면 안내가 사라집니다.
- **15초마다 서버와 대조**하는 안전망을 넣었습니다. 실시간 소식을 놓쳤어도 서버 상태가 더 최신이면 그때 따라잡습니다. 실시간이 완전히 죽어도 게임은 이어집니다.
- 다른 앱을 보다 돌아오면(`visibilitychange`) 즉시 맞춥니다.
- 방이 이미 삭제됐다면 로비로 내보냅니다.

**2. 방장이 사라져도 게임이 이어지도록**

- 진행이 **25초 넘게 멈추면** 그 방 참가자 중 **가장 먼저 들어온 사람**(방장 제외)이 진행을 이어받습니다. 컴퓨터 두기와 차례 넘기기 모두 가능합니다.
- 서버도 같은 기준(`_may_run`)으로 허용하므로, 클라이언트만 고친 게 아니라 권한 자체가 열립니다.
- 대행자는 4초 간격으로 시도해 서버에 부담을 주지 않습니다.
- 방장이 **나가기 버튼**을 누른 경우는 기존대로 즉시 승계됩니다. 이번 변경은 **창을 그냥 닫은 경우**를 위한 것입니다.

**3. 상태 위조 방지 (서버 검증)**

`push_state` 와 `force_state` 에서 `_check_state()` 로 아래를 확인합니다.

| 막는 것 | 메시지 |
|---|---|
| 다른 놀이의 상태 끼워넣기 | 다른 놀이의 상태예요 |
| 빽도·특별칸 설정 바꾸기 | 놀이 설정은 도중에 바꿀 수 없어요 |
| 참가자 추가·교체 | 참가자 정보는 바꿀 수 없어요 |
| 말 개수 늘리기 | 말 개수는 바꿀 수 없어요 |
| 완주 기록 되돌리기 | 완주 기록을 되돌릴 수 없어요 |
| 거짓으로 이겼다고 하기 | 아직 이긴 상태가 아니에요 |

**남은 한계**: 이동 규칙 전체를 서버에서 다시 계산하지는 않습니다. 말을 몇 칸 옮겼는지까지 검증하려면 서버에 규칙 엔진을 하나 더 두어야 해서, 가족·지인 규모에는 과합니다. 불특정 다수에게 공개한다면 그때 검토합니다.

**확인한 것**: 문법 검사, DOM 참조 검사, 600판 게임 시뮬레이션, `schema.sql` 구조 검사(구분자·begin/end 균형) 모두 통과.

### 2단계 · 중간 — 2026-07-22

**4. 채팅을 그 방 사람만 읽도록 (+ 7. 관전자 수 표시)**

관전자는 서버에 아무 기록이 없어서 "그 방 사람"인지 알 방법이 없었습니다. 그래서 참가자 표에 **역할(`role`)** 을 두고 관전자도 한 줄 기록하도록 바꿨습니다. 이 한 가지로 두 문제가 함께 풀립니다.

- `players.role` 추가 — `player`(놀이 참가) / `spectator`(관전)
- 채팅 읽기 정책을 **그 방에 기록이 있는 사람**으로 제한. 다른 방 대화는 API 로도 볼 수 없습니다
- 관전으로 들어가면 `watch_room()` 으로 기록되고, **관전 N명**이 방 화면과 놀이 화면에 표시됩니다
- 관전자가 팀을 고르면 그대로 참가자로 승격됩니다
- 정원(12명), 시작 조건, 방장 승계, 대행 자격은 모두 **실제 참가자만** 셉니다
- 관전자가 나가면 자기 기록만 지우고 게임에는 손대지 않습니다

**옛 스키마 호환**: 서버에 `role` 이 아직 없어도 깨지지 않습니다. 값이 없으면 참가자로 보고, `watch_room` 호출이 실패해도 조용히 넘어가 기존처럼 관전만 됩니다.

**5. 놀이 기록 전송량 절감**

기록 60줄을 매번 통째로 주고받던 것을, **새로 생긴 줄만** 실어 보내고 각자 화면에서 이어 쌓도록 바꿨습니다. 서버에는 새로 들어온 사람이 흐름을 파악할 만큼 **최근 6줄**만 남깁니다.

| | 이전 | 지금 |
|---|---|---|
| 상태 1회 전송 | 3.8 KB | **2.0 KB** (46% 절감) |
| 무료 5GB 기준 | 6,922판 | **12,874판** |

각 화면의 놀이 기록은 예전처럼 60줄까지 그대로 보입니다.

**확인한 것**: 문법 검사, DOM 참조 검사, 600판 시뮬레이션, 골인칸 선택 검사, `schema.sql` 구조 검사 모두 통과.

**8. 컴퓨터 실력 3단계**

방 설정과 혼자 연습 모두에서 **쉬움 · 보통 · 어려움**을 고를 수 있습니다.

- **쉬움** — 55% 확률로 최선이 아닌 수를 둡니다. 아이와 놀기 좋습니다
- **보통** — 기존 실력 (12% 정도만 흔들립니다)
- **어려움** — 흔들리지 않고, **다음 차례에 잡힐 자리를 피합니다**. 상대 말이 어떤 끗수로 내 자리에 닿을 수 있는지 확률로 계산합니다

900판씩 맞붙여 실력 순서를 확인했습니다.

| 대결 | 결과 |
|---|---|
| 어려움 vs 보통 | 53.9% |
| 보통 vs 쉬움 | 64.3% |
| 어려움 vs 쉬움 | 67.9% |

어려움과 보통의 차이(53.9%)는 실재하지만 크지는 않습니다. 위험 회피 가중치를 250~1300 까지 바꿔 봤으나 420~650 구간이 평평해 현재 값을 유지했습니다. 더 벌리려면 여러 수 앞을 내다보는 방식이 필요한데, 지금 구조에는 과합니다.

**시간 초과 자동 이동은 난이도와 무관하게 늘 최선의 수**를 둡니다. 내 차례를 컴퓨터가 대신 두면서 일부러 못 두면 곤란하니까요.

**9. 생각할 시간 설정**

방 설정에서 **20초 · 30초 · 60초 · 제한 없음**을 고를 수 있습니다. 제한 없음이면 남은 시간 표시와 자동 진행이 꺼지고, 방장의 "⏭ 넘기기" 버튼은 그대로 쓸 수 있습니다. 혼자 연습은 시간 제한이 없습니다.

**13. 오프라인 지원**

`sw.js` 를 추가해 앱 파일을 기기에 저장합니다. 지하철처럼 인터넷이 없는 곳에서도 화면이 뜨고 **혼자 연습**을 할 수 있습니다.

- **네트워크 우선** 방식이라 새로 배포한 내용이 바로 반영됩니다. 실패할 때만 저장해 둔 파일을 씁니다
- 서버(Supabase)와 글꼴 요청은 저장하지 않습니다
- 인터넷이 없으면 로비에 **✈️ 안내줄**이 뜨고 방 만들기는 잠깁니다. 프로필은 기기에만 저장됩니다
- 다시 연결되면 알려 주고, 연습 중이 아니면 자동으로 새로고침해 온라인 기능을 되살립니다

### 14. 게임 중 감정 표현 — 2026-07-22

머리말의 **😀 버튼**을 누르면 아래쪽에 이모티콘 줄이 열립니다. 하나를 고르면 화면 아래에서 **위로 떠오르며 사라지는** 표현이 보내는 사람 이름과 함께 모두의 화면에 뜹니다. 👍 👏 🎉 😂 😮 😭 😡 🙏 여덟 가지입니다.

**저장하지 않습니다.** 잠깐 떴다 사라지는 표현이라 DB 대신 실시간 전송(broadcast)으로 전달합니다. 대화 기록처럼 쌓이지 않아 용량도 정리 부담도 없습니다.

**말판을 가리지 않도록 세 군데에서 제한**했습니다.

| 어디서 | 무엇을 |
|---|---|
| 보내는 쪽 | 2초에 한 번, 15초 안에 5번까지. 넘으면 8초 잠금 |
| 받는 쪽 | 같은 사람이 1.2초 안에 또 보내면 무시 |
| 화면 | 동시에 최대 10개까지만 표시 |

받는 쪽에도 제한을 둔 이유는, 프로그램을 고쳐 마구 보내는 사람이 있어도 **내 화면은 지켜지도록** 하기 위해서입니다.

실제 동작을 모사해 확인했습니다.

| 사용 방식 | 결과 |
|---|---|
| 0.1초 간격 연타 (3초간) | 2번만 전송 |
| 2.1초마다 꾸준히 (30초간) | 10번 전송, 5번 막힘 |
| 5초마다 여유있게 (60초간) | 12번 전송, **한 번도 안 막힘** |

보통 속도로 쓰면 걸리지 않고 연타만 막힙니다. 잠금 중에는 버튼이 흐려지고 남은 시간이 막대로 보이며, 이모티콘 줄은 쓰지 않으면 몇 초 뒤 저절로 닫혀 화면을 가리지 않습니다.
