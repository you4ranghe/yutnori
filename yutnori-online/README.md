# 윷놀이 온라인 🎲

로비에서 열린 방을 눌러 바로 들어가 함께 즐기는 실시간 윷놀이입니다.
혼자서 컴퓨터와 연습할 수도 있어요. Supabase(무료) + Vercel(무료)만으로 운영됩니다.

## 기능

- **로비** — 열린 방 목록과 지금 접속 중인 사람이 한눈에. 방을 누르면 바로 입장 (초대 코드 불필요)
- **방 화면** — 방을 만들면 팀을 고르며 친구를 기다릴 수 있어요. **빈 팀은 컴퓨터가 맡아** 혼자여도 바로 시작 가능
- **친구가 오면 알림** — 컴퓨터와 놀던 중 누가 방에 들어오면 "함께 놀까요?" 알림이 뜨고, 한 번 눌러 대기방으로 돌아가 같이 시작할 수 있어요 (원할 때 직접 돌아가는 버튼도 있음)
- **관전** — 이미 시작된 게임은 참가 대신 관전으로 들어가 구경할 수 있어요
- **혼자 연습하기** — 🐢 나 vs 🐰 컴퓨터. 연습 중 누가 방을 만들면 알림이 떠서 바로 합류
- **로그인 · 프로필** — 아이디 + 비밀번호로 **이메일 인증 없이** 가입 즉시 로그인. 이름·프로필 사진이 계정에 저장되어 다른 기기에서도 그대로 (가입 없이 게스트로도 가능)
- **채팅** — 게임 중 실시간 대화 (관전자도 참여 가능)
- **20초 자동 진행** — 시간 안에 던지지/움직이지 않으면 자동 진행
- **중간 이탈 처리** — 나간 사람의 팀에 아무도 없으면 상대 팀 승리, 한 명이라도 남으면 2:1로 계속

## 구성 파일

| 파일 | 역할 |
|---|---|
| `index.html` | 로비 · 게임 · 채팅 · 프로필 전체 화면 |
| `config.js` | Supabase 접속 정보 (직접 입력 필요) |
| `schema.sql` | Supabase에 실행하는 DB 스키마 + 서버 함수 |

## 배포 순서 (약 15분)

### 1. Supabase 프로젝트 만들기

1. https://supabase.com 접속 → 가입/로그인 → **New Project**
2. 프로젝트 이름(예: `yutnori`), DB 비밀번호, 리전(**Northeast Asia (Seoul)** 권장) 선택 → 생성

### 2. DB 스키마 설치

1. 대시보드 왼쪽 메뉴 → **SQL Editor**
2. `schema.sql` 파일 내용 **전체**를 붙여넣고 **Run**
3. "Success" 가 나오면 완료 (여러 번 실행해도 안전합니다)

> 기능이 추가되어 스키마가 바뀌면 `schema.sql`을 **다시 한 번 실행**하세요. 기존 데이터는 유지됩니다.

### 3. 로그인 방식 켜기 ⚠️ 필수

**Authentication → Sign In / Providers** 에서:

1. **Anonymous sign-ins** 를 **켜기(Enable)** → **Save changes** 클릭
   - 회원가입 없이 게스트로 바로 놀 수 있게 해줍니다. 토글만 켜고 저장을 누르지 않으면 적용되지 않아요.
2. **Email** 을 펼쳐서 **Confirm email** 을 **끄기(Disable)** → **Save changes** 클릭
   - 이 게임은 이메일 인증 없이 **아이디 + 비밀번호**로 가입하고 즉시 로그인되도록 만들어져 있습니다.
   - 이 설정이 켜져 있으면 가입해도 로그인이 되지 않고 "이메일 확인이 켜져 있어요" 안내가 뜹니다.
   - 아이디는 내부적으로 `아이디@yutnori.app` 형태로 저장되며 실제 메일은 오가지 않습니다.

> 설정이 제대로 적용됐는지는 브라우저에서 아래 주소를 열어 확인할 수 있어요.
> `https://<프로젝트>.supabase.co/auth/v1/settings?apikey=<anon키>`
> `mailer_autoconfirm` 이 **true**, `external.anonymous_users` 가 **true** 여야 정상입니다.

### 4. 접속 정보 입력

1. 대시보드 → **Project Settings** → **API Keys**
2. `config.js` 에 두 값을 채워 넣기:
   - **Project URL** → `SUPABASE_URL`
   - **anon public** 키(또는 **Publishable key**) → `SUPABASE_ANON_KEY`
   - 이 키는 공개되어도 되는 키입니다 (권한은 DB의 RLS가 통제).

### 5. Vercel에 배포

방법 A — CLI:
```
cd yutnori-online
npx vercel          # 미리보기
npx vercel --prod   # 정식 배포
```

방법 B — GitHub 연동:
1. https://vercel.com → **Add New Project** → 저장소 선택
2. **Root Directory** 를 `yutnori-online` 으로 지정
3. Framework Preset: **Other** → **Deploy**

로컬에서 먼저 확인하려면 `npx serve` 로 여세요 (`file://` 직접 열기는 동작하지 않습니다).
설정을 바꾼 뒤에는 **Ctrl+F5** 로 강력 새로고침하세요.

## 노는 방법

1. 접속하면 바로 **로비**가 열립니다. 원하면 **✏️ 프로필**에서 이름·사진을 설정하세요.
2. **🏠 방 만들기** 로 방을 열면, 다른 사람의 로비 목록에 즉시 나타납니다.
3. 목록에서 방을 누르면 입장 — **모집 중**이면 참가, **놀이 중**이면 관전으로 들어갑니다.
4. 두 팀에 각각 1명 이상 모이면 방장이 **놀이 시작**.
5. 자기 차례인 사람만 던지고 옮길 수 있어요. 💬 버튼으로 채팅도 가능합니다.

혼자 놀려면 **🎯 혼자 연습** 을 누르세요. 연습 중 누군가 방을 만들면 알림이 뜨고, **참가하기** 로 바로 합류합니다.

## 운영 관련 메모

- **데이터는 쌓이지 않습니다.** **1시간 동안 아무 진행이 없는 방은 자동으로 삭제**됩니다(던지기·말 이동·참가/퇴장·채팅이 있으면 시간이 초기화). 어떤 방이든 24시간이 지나면 삭제됩니다. 채팅은 방마다 최근 100개만 남고 방과 함께 지워집니다.
  - 청소는 두 가지 방법으로 돕니다: ① 누군가 로비를 열 때(최대 1분에 한 번), ② `pg_cron` 이 설치되면 10분마다 자동으로. `schema.sql` 이 pg_cron 예약을 시도하고, 권한이 없으면 조용히 건너뜁니다.
  - 방이 삭제되면 그 방에 있던 사람은 곧바로 로비로 돌아가고, 나중에 다시 접속해도 로비에서 시작합니다.
- **프로필 사진**은 128×128로 줄여 약 10KB로 저장되므로 무료 용량에 부담이 없습니다.
- **윷 결과는 서버(DB 함수)에서 생성**되므로 클라이언트 조작으로 원하는 끗수를 만들 수 없습니다.
- **재접속 지원**: 새로고침하거나 잠깐 끊겨도 같은 브라우저로 다시 열면 하던 방(관전 포함)으로 돌아갑니다.
- **자리 비움 대비**: 20초 뒤 자동 진행, 그래도 응답이 없으면 32초 뒤 방장 화면에서 자동으로 차례를 넘깁니다.
- **방장이 나가면** 남은 사람 중 가장 먼저 들어온 사람이 방장을 이어받습니다.
- **Supabase 무료 프로젝트는 1주일간 요청이 없으면 일시정지**됩니다. 대시보드의 Restore 버튼으로 재개하세요.
- Vercel 무료(Hobby) 플랜은 비상업 용도 조건이 있습니다. 지인들과의 게임 용도는 문제없습니다.

## 알려진 한계

- 진행 중인 게임에는 참가할 수 없고 관전만 가능합니다 (게임이 끝나면 참가할 수 있어요).
- 방 목록과 접속자 목록은 접속한 모든 사람에게 보입니다. 완전한 비공개 방은 지원하지 않습니다.
- 게스트(비로그인) 계정의 프로필은 그 기기에만 남습니다. 다른 기기에서도 쓰려면 회원가입하세요. 게스트로 쓰던 이름과 사진은 가입할 때 새 계정으로 자동으로 옮겨집니다.
- 비밀번호 찾기 기능은 없습니다(실제 이메일을 쓰지 않기 때문). 잊어버리면 새 아이디로 가입해야 합니다.
- 브라우저를 그냥 닫으면(나가기 버튼 미사용) 이탈이 즉시 반영되지 않습니다. 이 경우 20초 자동 진행과 방장의 차례 넘기기로 게임이 계속됩니다.
