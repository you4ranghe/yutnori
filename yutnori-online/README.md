# 윷놀이 온라인 🎲

친구들과 각자 폰·PC에서 초대 코드로 모여 함께 즐기는 실시간 윷놀이입니다.
혼자서 컴퓨터와 연습할 수도 있어요. Supabase(무료) + Vercel(무료)만으로 운영됩니다.

## 기능

- **온라인 대전** — 초대 코드(또는 링크)로 최대 12명이 두 팀으로 나눠 실시간 대결
- **혼자 연습하기** — 컴퓨터와 한 판. 연습 중에 누가 방을 만들면 알림이 떠서 바로 참가 가능
- **로그인 · 프로필** — 이름과 프로필 사진을 저장해 게임 중 표시 (가입 없이 게스트로도 가능)
- **채팅** — 게임 중 실시간 대화
- **20초 자동 진행** — 시간 안에 던지지/움직이지 않으면 자동으로 진행 (자리를 비워도 게임이 멈추지 않음)

## 구성 파일

| 파일 | 역할 |
|---|---|
| `index.html` | 게임 화면 전체 (로비 → 대기실 → 게임 · 채팅 · 프로필) |
| `config.js` | Supabase 접속 정보 (직접 입력 필요) |
| `schema.sql` | Supabase에 실행하는 DB 스키마 + 서버 함수 |

## 배포 순서 (약 15분)

### 1. Supabase 프로젝트 만들기

1. https://supabase.com 접속 → 가입/로그인 → **New Project**
2. 프로젝트 이름(예: `yutnori`), DB 비밀번호, 리전(**Northeast Asia (Seoul)** 권장) 선택 → 생성

### 2. DB 스키마 설치

1. 대시보드 왼쪽 메뉴 → **SQL Editor**
2. `schema.sql` 파일 내용 **전체**를 붙여넣고 **Run**
3. "Success" 가 나오면 완료 (여러 번 실행해도 안전합니다)

> 기능이 추가되어 스키마가 바뀌었다면 `schema.sql`을 **다시 한 번 실행**하세요. 기존 데이터는 그대로 유지됩니다.

### 3. 로그인 방식 켜기

**Authentication → Sign In / Providers** 에서:

1. **Anonymous sign-ins** 를 **켜기(Enable)** → **Save changes** 클릭
   - 회원가입 없이 게스트로 바로 놀 수 있게 해줍니다. 토글만 켜고 저장을 누르지 않으면 적용되지 않아요.
2. **Email** 이 켜져 있는지 확인 (회원가입/로그인용)
   - 같은 화면의 **Confirm email** 을 **끄면** 가입 즉시 바로 놀 수 있습니다. 켜 두면 가입 후 메일의 확인 링크를 눌러야 로그인됩니다.

### 4. 접속 정보 입력

1. 대시보드 → **Project Settings** → **API Keys**
2. `config.js` 를 열어 두 값을 채워 넣기:
   - **Project URL** → `SUPABASE_URL`
   - **anon public** 키(또는 **Publishable key**) → `SUPABASE_ANON_KEY`
   - 이 키는 공개되어도 되는 키입니다 (권한은 DB의 RLS가 통제).

### 5. Vercel에 배포

방법 A — CLI:
```
cd yutnori-online
npx vercel
```
안내에 따라 로그인 → 기본값으로 배포 → 나오는 URL이 게임 주소입니다.
`npx vercel --prod` 로 정식 주소에 올립니다.

방법 B — GitHub 연동:
1. https://vercel.com → **Add New Project** → 저장소 선택
2. **Root Directory** 를 `yutnori-online` 으로 지정
3. Framework Preset: **Other** → **Deploy**

로컬에서 먼저 확인하려면:
```
cd yutnori-online
npx serve
```
(`file://` 로 직접 열면 동작하지 않으니 로컬 서버로 여세요. 설정을 바꾼 뒤에는 **Ctrl+F5** 로 강력 새로고침하세요.)

## 노는 방법

**함께 놀기**
1. 접속 → (원하면 프로필에서 이름·사진 설정) → **방 만들기**
2. **초대 코드**(또는 링크)를 친구에게 전송. 로비의 "지금 열린 방" 목록에서도 보여요
3. 친구들이 참가해 팀을 고르면, 방장이 옵션을 정하고 **놀이 시작**
4. 자기 차례인 사람만 던지고 옮길 수 있어요. 💬 버튼으로 채팅도 가능

**혼자 놀기**
- 로비에서 **혼자 연습하기 🎯** → 컴퓨터와 대결
- 연습 중에 누군가 방을 만들면 화면 위에 알림이 뜨고, **참가하기** 를 누르면 바로 합류합니다

## 운영 관련 메모

- **데이터는 쌓이지 않습니다.** 끝난 방은 1시간 뒤, 모든 방은 24시간 뒤 자동 삭제됩니다 (새 방이 만들어질 때마다 청소 실행). 채팅은 방마다 최근 100개만 남고 방과 함께 지워집니다.
- **프로필 사진**은 128×128로 줄여 약 10KB로 저장되므로 무료 용량에 부담이 없습니다.
- **윷 결과는 서버(DB 함수)에서 생성**되므로 클라이언트 조작으로 원하는 끗수를 만들 수 없습니다.
- **재접속 지원**: 새로고침하거나 잠깐 끊겨도 같은 브라우저로 다시 열면 하던 방으로 돌아갑니다.
- **자리 비움 대비**: 20초가 지나면 자동으로 던지고/움직입니다. 그래도 응답이 없으면(접속 종료 등) 32초 뒤 방장 화면에서 자동으로 차례를 넘깁니다. 방장은 언제든 "⏭ 넘기기" 버튼을 쓸 수 있어요.
- **Supabase 무료 프로젝트는 1주일간 요청이 없으면 일시정지**됩니다. 대시보드에서 Restore 버튼으로 재개하면 됩니다.
- Vercel 무료(Hobby) 플랜은 비상업 용도 조건이 있습니다. 지인들과의 게임 용도는 문제없습니다.

## 알려진 한계

- 게임이 시작된 방에는 새 사람이 들어올 수 없습니다 (시작 전 대기실에서만 참가 가능).
- "지금 열린 방" 목록은 접속한 모든 사람에게 보입니다. 비공개로 놀고 싶다면 초대 코드만 공유하고 상대가 링크로 바로 들어오게 하세요.
- 게스트(비로그인) 계정의 프로필은 그 기기에만 남습니다. 다른 기기에서도 쓰려면 회원가입하세요.
