# 검사 도구

코드를 고친 뒤 **한 줄로** 이상이 없는지 확인합니다. Node 만 있으면 됩니다.

```
cd yutnori-online
node tools/검사.js
```

전부 통과하면 `✅ 전부 통과`, 하나라도 어긋나면 무엇이 문제인지 찍고 종료 코드 1 로 끝납니다.

## 무엇을 보는가

| 항목 | 잡아내는 것 |
|---|---|
| **문법** | `index.html` 안의 게임 코드, `config.js`, `sw.js` 의 문법 오류 |
| **화면 요소** | 코드가 찾는 `id`·클래스가 화면에 실제로 있는지. 화면 구조를 크게 바꾼 뒤 필수 |
| **스키마** | `$$` 구분자 짝, 트랜잭션 여부, 함수 `begin/end` 균형, 공개 함수 권한 누락 |
| **게임 규칙** | 600판을 끝까지 돌려 멈추거나 둘 수 없는 상황이 없는지, 선공 유불리, 출발·도착 칸 선택, 컴퓨터 실력 사다리 |

## 한계

- **스키마는 실제로 실행해 보지 않습니다.** 구분자나 괄호 짝 같은 겉모습만 봅니다. 컬럼명 오타나 타입 오류는 Supabase SQL Editor 에서 돌려야 드러납니다.
- 화면에 실제로 그려지는 모습(색·크기·배치)은 확인하지 않습니다. 눈으로 봐야 합니다.
- 서버와 주고받는 실제 동작(실시간, 권한)은 확인하지 않습니다.

## 파일

- `검사.js` — 전체 검사. 보통 이것만 실행하면 됩니다
- `lib.js` — `index.html` 에서 게임 코드를 꺼내 오는 공용 부분

`lib.js` 는 **최상위 함수의 닫는 중괄호가 열 0 에 있다**는 코드 모양에 기대어 함수를 꺼냅니다. 그 규칙을 깨면 검사가 함수를 못 찾습니다.
